# BrainTwist 开发操作清单

## 1. 开发前检查

每次开始开发前先确认：

- 当前目录是 Cocos 项目根目录。
- `package.json` 中 Cocos Creator 版本为 `3.8.8`。
- `assets/` 目录存在。
- Lottie 动画位于 `assets/resources/animations/lottie/`。
- 文档位于 `docs/`。

当前已规划的动画文件：

```text
assets/resources/animations/lottie/arrow-right-click.json
```

## 2. Codex 本地文件操作

Codex 可以直接完成这些本地工作：

- 新建脚本文件。
- 修改 TypeScript 游戏逻辑。
- 整理资源目录。
- 编写文档和任务列表。
- 检查文件路径、命名和项目状态。
- 执行非破坏性的终端命令。

建议优先让 Codex 处理代码和文档，再用 Cocos Creator 编辑器完成节点绑定和可视化调整。

## 3. 使用 Computer Use 操作 Cocos

当需要点击 Cocos Creator 图形界面时，可以让 Codex 通过 `Computer Use` 操作本机。

适合交给 `Computer Use` 的任务：

- 打开 Cocos Creator。
- 打开当前项目。
- 打开当前 `assets/scene.scene`，或在 Cocos Creator 中重命名整理为 `assets/scenes/Main.scene`。
- 在层级面板创建 Canvas、Label、Button、Sprite 等节点。
- 给节点挂载 `GameController`、`DirectionButton` 等脚本。
- 在 Inspector 中拖拽绑定属性。
- 给四个按钮绑定点击事件。
- 运行预览并检查画面反馈。

建议的 Cocos 场景节点结构：

```text
Canvas
  GameRoot
    ScoreLabel
    ArrowDisplay
    FeedbackAnimation
    DirectionButtons
      ButtonUp
      ButtonDown
      ButtonLeft
      ButtonRight
    GameOverPanel
      FinalScoreLabel
      RestartButton
```

按钮方向绑定建议：

- `ButtonUp` -> `Direction.Up`
- `ButtonDown` -> `Direction.Down`
- `ButtonLeft` -> `Direction.Left`
- `ButtonRight` -> `Direction.Right`

## 4. 使用 lottiefiles-creator 创建动画

`lottiefiles-creator` 适合用于创建或修改游戏反馈动画。

当前已有动画：

- 名称：箭头向右动画
- 文件：`arrow-right-click.json`
- 用途：点击后触发的右向箭头反馈

后续动画建议：

```text
arrow-up-click.json
arrow-down-click.json
arrow-left-click.json
arrow-right-click.json
```

如果只保留一个右向动画，优先通过旋转节点复用，减少资源数量和维护成本。

动画导入流程：

1. 通过 `lottiefiles-creator` 创建或编辑动画。
2. 导出 JSON 文件。
3. 放入 `assets/resources/animations/lottie/`。
4. 在 Cocos Creator 中等待资源导入。
5. 在脚本或组件中按资源路径加载。
6. 在按钮点击成功或失败时触发播放。

## 5. 最小可玩版本开发顺序

1. 创建 `assets/scripts/Direction.ts`。
2. 创建 `assets/scripts/GameController.ts`。
3. 创建 `assets/scripts/DirectionButton.ts`。
4. 在 Cocos 中打开当前 `assets/scene.scene`，或重命名整理为 `assets/scenes/Main.scene`。
5. 创建中央箭头显示节点。
6. 创建四个方向按钮。
7. 绑定按钮点击事件到 `GameController`。
8. 实现随机箭头刷新。
9. 实现点击判定。
10. 实现分数显示。
11. 实现游戏结束和重新开始。
12. 接入 `arrow-right-click.json` 动画反馈。

## 6. 点击判定标准

当前版本判定：

```text
clickedDirection !== arrowDirection
```

示例：

- 当前箭头朝右，点击右：失败。
- 当前箭头朝右，点击上：成功。
- 当前箭头朝右，点击下：成功。
- 当前箭头朝右，点击左：成功。

如果后续改为严格反向规则，判定改为：

```text
clickedDirection === oppositeOf(arrowDirection)
```

示例：

- 当前箭头朝右，只有点击左成功。
- 当前箭头朝上，只有点击下成功。

## 7. 调试检查点

每完成一个阶段检查一次：

- 启动场景时是否立即显示箭头。
- 箭头方向是否随机变化。
- 点击同方向是否结束游戏。
- 点击不同方向是否加分并刷新。
- 游戏结束后按钮是否停止继续加分。
- 重新开始是否重置分数和状态。
- 动画资源是否能正常加载。
- 预览窗口是否有报错。

## 8. 后续增强方向

可在最小可玩版本稳定后加入：

- 倒计时压力。
- 连击加分。
- 错误震屏或红色闪烁。
- 成功点击时播放箭头飞出动画。
- 难度随分数提升。
- 背景音乐和点击音效。
- 本地最高分。
- 移动端触控适配。
- 微信小游戏或 Web 构建配置。

## 9. 操作边界

以下操作需要明确确认后再执行：

- 删除或覆盖现有资源。
- 安装插件或依赖。
- 修改 Cocos 全局设置。
- 登录、发布、上传或构建到外部平台。
- 对外推送代码或创建远程仓库。

日常代码、文档、目录整理和本地预览调试可以直接推进。
