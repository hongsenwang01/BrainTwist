# 最高分接口接入说明

## 前端位置

首页最高分节点：`assets/游戏首页.scene` 里的 `02-05最高分/分数`

当前该节点挂载的是 `RandomScoreLabel`。已调整为：

- `randomizeOnStart = false`
- `fetchBestScoreOnStart = true`
- `bestScoreApiPath = /api/game-scores/best`
- `gameKey = reverse_brain`
- `gameMode = classic`

进入首页后会先读本地缓存 `brain_twist_best_score_cache_v1`，再按需请求最高分接口。游戏总结上传成功后，如果后端返回 `bestScore`，前端会同步更新这个缓存。

## 请求接口

```http
GET /api/game-scores/best?userId={userId}&gameKey=reverse_brain&gameMode=classic&difficulty=normal
```

字段说明：

| 字段 | 位置 | 必填 | 说明 |
| --- | --- | --- | --- |
| userId | query | 是 | 登录接口返回并写入本地的用户 ID |
| gameKey | query | 是 | 当前为 `reverse_brain` |
| gameMode | query | 是 | 当前为 `classic` |
| difficulty | query | 否 | 当前普通模式传 `normal`；不传时返回该用户、游戏、模式下所有难度中的最高分 |

成功响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "bestScore": 464
  }
}
```

失败响应：

```json
{
  "code": 10001,
  "message": "userId is required"
}
```

## 后端实现要求

该接口必须从“用户最高分/统计表”读取，不要从对局记录表 `game_score_records` 做 `MAX(score)`。

原因：对局记录表是明细流水，可能会被清理、归档、删除测试数据；用户历史最高分应该以专门维护的最高分表为准。否则删除对局记录后，前端会拿到清理后的明细最大值，例如只剩 37，但最高分表里实际是 202。

推荐查询：

```ts
app.get("/api/game-scores/best", async (req, res) => {
  const { userId, gameKey = "reverse_brain", gameMode = "classic" } = req.query;

  if (!userId) {
    return res.json({ code: 10001, message: "userId is required" });
  }

  const row = await db.queryOne(`
    SELECT COALESCE(best_score, 0) AS bestScore
    FROM user_game_stats
    WHERE user_id = ?
      AND game_key = ?
      AND game_mode = ?
      AND (? IS NULL OR difficulty = ?)
  `, [userId, gameKey, gameMode, difficulty ?? null, difficulty ?? null]);

  return res.json({
    code: 0,
    message: "ok",
    data: {
      bestScore: Number(row?.bestScore ?? 0)
    }
  });
});
```

建议索引：

```sql
CREATE UNIQUE INDEX uk_user_game_stats_best
ON user_game_stats (user_id, game_key, game_mode);
```

如果最高分表字段名不同，保持接口响应字段仍为 `data.bestScore` 即可：

```sql
SELECT best_score AS bestScore
FROM user_game_stats
WHERE user_id = ?
  AND game_key = ?
  AND game_mode = ?;
```

## 和游戏总结接口的关系

已有：

```http
POST /api/game-scores/submit
```

该接口建议继续返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "roundId": "round_001",
    "isNewBest": true,
    "bestScore": 464
  }
}
```

这里的 `bestScore` 必须是“提交本局成绩后的最终历史最高分”：

- 如果本局是新纪录：`bestScore = 本局分数`
- 如果本局不是新纪录：`bestScore = 原历史最高分`

前端会在收到 `bestScore` 后写入本地缓存。这样玩家打完一局回到首页时，即使最高分接口还没请求完成，也能先看到最新最高分。

游戏总结页的“历史最高分”显示的是“本局提交前已有记录的最高分”，不包含本局成绩。流程是：

1. 进入游戏总结页，先用本地缓存显示一个历史最高分。
2. 在提交本局成绩前，请求 `GET /api/game-scores/best`，拿当前数据库里已有记录的最高分并更新总结页文本。
3. 然后再调用 `POST /api/game-scores/submit` 提交本局成绩。
4. `POST /api/game-scores/submit` 返回的 `bestScore` 只用于更新本地缓存和首页显示，不再覆盖游戏总结页的历史最高分文本。
