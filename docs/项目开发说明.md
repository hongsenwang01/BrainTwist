# BrainTwist 项目开发说明

## 项目定位

BrainTwist 是一个“反向大脑”反应游戏。玩家需要根据屏幕上随机刷新的箭头做出反向判断，核心挑战不是速度本身，而是在短时间内抑制直觉点击。

当前项目使用 Cocos Creator 3.8.8 创建，后续主要使用 TypeScript 编写游戏逻辑，并通过 Cocos 编辑器完成场景、节点、组件、资源绑定和预览调试。

## 核心玩法

1. 屏幕中央随机刷新一个箭头方向。
2. 玩家需要点击方向按钮。
3. 如果点击方向与箭头指向相同，游戏立即结束。
4. 如果点击方向与箭头指向不同，本轮通过并刷新下一个箭头。
5. 分数随连续通过次数增长。
6. 后续可以增加倒计时、连击、难度递增、音效、动画反馈和排行榜。

当前规则按“不能点击箭头指向方向”实现：例如箭头朝右时，点击右侧失败，点击上、下、左任意方向成功。后续如果想强化成“必须点击严格相反方向”，只需要把判定逻辑从 `clickedDirection !== arrowDirection` 调整为 `clickedDirection === oppositeOf(arrowDirection)`。

## 方向数据

建议统一用枚举维护方向，避免在各脚本里散落字符串。

```ts
export enum Direction {
  Up = "up",
  Down = "down",
  Left = "left",
  Right = "right",
}
```

推荐的基础函数：

- `randomDirection()`：随机生成当前箭头方向。
- `isValidClick(arrowDirection, clickedDirection)`：判断本轮点击是否通过。
- `oppositeOf(direction)`：后续严格反向模式会用到。

## 推荐目录规划

```text
BrainTwist/
  assets/
    scene.scene
    scenes/
      Main.scene
    scripts/
      Direction.ts
      GameController.ts
      DirectionButton.ts
      LottieAnimationPlayer.ts
    prefabs/
      DirectionButton.prefab
    resources/
      animations/
        lottie/
          arrow-right-click.json
    textures/
      arrows/
  docs/
    项目开发说明.md
    开发操作清单.md
  settings/
  package.json
  tsconfig.json
```

说明：

- `assets/scene.scene`：当前 Cocos 已生成的默认场景，前期可以直接作为主场景使用。
- `assets/scenes/`：放主场景，例如 `Main.scene`。
- `assets/scripts/`：放游戏逻辑脚本。
- `assets/prefabs/`：放可复用按钮、箭头或反馈节点。
- `assets/resources/animations/lottie/`：放运行时需要动态加载的 Lottie JSON 动画。
- `assets/textures/arrows/`：放静态箭头图片、按钮贴图和 UI 图形资源。
- `docs/`：放开发说明、操作流程、设计记录。

当前建议先不要手动移动 `.scene` 和 `.meta` 文件。正式搭建场景时，可通过 Cocos Creator 编辑器把默认场景重命名或整理到 `assets/scenes/Main.scene`，这样能让编辑器同步维护资源 UUID 和 `.meta` 关系。

## 当前动画资源

已创建的 Lottie 动画文件：

```text
assets/resources/animations/lottie/arrow-right-click.json
```

来源说明：

- 原始文件名：`箭头向右动画.json`
- 当前规划用途：点击右方向或箭头飞出反馈动画。
- 创建工具：`lottiefiles-creator` 本机 MCP 服务。
- 动画内容：右向箭头点击后的动态反馈。

运行时建议加载路径：

```ts
resources.load("animations/lottie/arrow-right-click", JsonAsset, callback);
```

如果后续需要上、下、左三个方向动画，建议用同一套命名：

```text
arrow-up-click.json
arrow-down-click.json
arrow-left-click.json
arrow-right-click.json
```

如果只维护一个右向动画，也可以在 Cocos 中通过节点旋转复用：

- 右：`0`
- 下：`90`
- 左：`180`
- 上：`-90`

## Cocos 组件规划

### GameController

负责游戏主流程：

- 初始化游戏状态。
- 随机生成箭头方向。
- 更新箭头显示。
- 接收方向按钮点击。
- 判断成功或失败。
- 维护分数、连击和游戏结束状态。
- 触发成功、失败、刷新动画。

### DirectionButton

挂在四个方向按钮上：

- 保存按钮代表的方向。
- 点击后通知 `GameController`。
- 只处理输入，不直接决定游戏结果。

### LottieAnimationPlayer

负责播放 Lottie JSON 动画：

- 加载 `resources/animations/lottie/` 下的 JSON。
- 根据当前方向旋转或替换动画。
- 播放点击反馈、成功反馈或失败反馈。

Cocos 原生不直接等价于 Web 的 lottie-web，需要在正式实现前确认渲染方案：

- 方案一：用 Cocos 支持的第三方 Lottie 组件或插件。
- 方案二：将 Lottie 导出为序列帧或 Sprite 动画。
- 方案三：把 Lottie 动画作为开发参考，在 Cocos 中重建为 Tween 动画。

## Codex 可协助的工作

Codex 可以在项目目录内直接完成：

- 创建和整理目录结构。
- 编写 TypeScript 脚本。
- 创建开发文档、需求说明和任务清单。
- 检查项目配置文件。
- 规划 Cocos 节点结构、资源命名和脚本职责。
- 运行本机命令做检查，例如 `git status`、文件搜索、构建脚本等。
- 根据后续实现结果修复逻辑错误和类型问题。

通过 `Computer Use` 插件，Codex 还可以操作本机图形界面中的 Cocos Creator，例如：

- 打开 Cocos Creator 项目。
- 点击创建场景、节点、按钮和 UI 层级。
- 给节点添加脚本组件。
- 在 Inspector 面板里绑定按钮事件、资源引用和组件属性。
- 启动预览，观察游戏流程是否符合预期。
- 调整项目设置、构建面板和编辑器配置。

涉及删除资源、覆盖文件、发布、登录账号、联网下载依赖或对外上传时，需要先明确确认。

通过 `lottiefiles-creator` MCP 服务，Codex 可以继续协助：

- 创建新的箭头方向动画。
- 调整已有 Lottie 动画的尺寸、帧率、颜色和节奏。
- 导出适合 Cocos 使用的 JSON 文件。
- 规划动画命名、版本和触发时机。

## 第一阶段开发目标

第一阶段只做最小可玩版本：

1. 使用当前 `assets/scene.scene` 作为主场景，或在 Cocos Creator 中重命名为 `assets/scenes/Main.scene`。
2. 放置一个中央箭头显示节点。
3. 放置上、下、左、右四个点击按钮。
4. 编写 `GameController.ts`。
5. 点击正确方向后刷新箭头并加分。
6. 点击箭头同方向后进入结束状态。
7. 接入右向点击动画，先验证资源加载和播放路径。

第一阶段完成后，再逐步加入动画润色、音效、倒计时、难度曲线和结算界面。
