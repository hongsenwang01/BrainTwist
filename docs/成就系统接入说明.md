# 成就系统接入说明

## 前端现状

成就页脚本：`assets/scripts/成就/AchievementSceneView.ts`

本次已把成就列表改成数据驱动：

- 本地默认只保留 1 条成就：`初来乍到`
- 页面启动后会读取 `brain_twist_user_id`
- 页面会先读取本地缓存并立即渲染，不会每次打开都强制等待接口
- 无缓存、缓存过期、或游戏总结上传成功后标记缓存过期时，才请求后端成就列表接口
- 后端返回多少条，页面就按同一个行渲染逻辑向下列出多少条
- 页面会保留 Cocos 场景里已经摆好的静态 UI，优先复用一个通用模板节点 `成就条模板` 生成列表；如果没有这个节点，会兼容旧的 `成就条-一击即中` 示例行作为模板
- 成就列表运行时会放进脚本创建的 `成就列表视窗`，列表超出可视区域时上下滑动，返回按钮保持固定不动
- 请求失败时默认显示本地 `初来乍到`，便于后端未完成时继续调 UI

## 缓存策略

当前采用“本地缓存 + 脏标记 + 短 TTL”的方式，避免频繁请求接口：

1. 成就页打开时，优先读取 `localStorage.brain_twist_achievement_cache_v1`，有缓存就立即渲染。
2. 如果没有缓存，或者缓存超过 `缓存有效期秒`，才请求 `/api/achievements`。
3. 游戏总结上传成功后，会写入 `localStorage.brain_twist_achievement_cache_dirty = "1"`。
4. 下次打开成就页时，如果发现 dirty 标记，会请求一次后端，刷新本地缓存并清除 dirty 标记。
5. 如果玩家只是反复打开成就页，没有打完新一局，也没有超过缓存时间，就不会重复请求接口。

这样能达到两个目标：

- 成就页打开快：先展示本地缓存，不等接口。
- 进度足够实时：每局游戏总结上传成功后，下次打开成就页一定刷新一次。

如果后续希望“游戏总结页立刻弹出新解锁成就”，可以让 `/api/game-scores/submit` 在响应里额外返回 `changedAchievements`，前端直接更新缓存并展示弹窗；成就页仍然使用当前缓存机制。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 成就唯一 key，例如 `tutorial` |
| category | string | 是 | `growth`、`challenge`、`reaction`、`collect`、`hidden` |
| title | string | 是 | 成就名称 |

| current | number | 是 | 当前进度 |
| target | number | 是 | 目标进度，最小为 1 |
| progressText | string | 否 | 展示文本，例如 `6/10h`；不传则前端显示 `current/target` |
| unlocked | boolean | 是 | 是否已解锁 |
| showProgress | boolean | 否 | 是否显示进度条；不传时未解锁默认显示 |

```json
{
  "code": 10001,
  "message": "userId is required"
}
```

## MySQL 建表建议

如果后端使用 MySQL 8，可以先建这两张表。成就定义和用户进度分开，方便后续新增成就，不需要改前端。

```sql
CREATE TABLE achievement_definitions (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY,
  achievement_key VARCHAR(64) NOT NULL,
  game_key VARCHAR(64) NOT NULL DEFAULT 'reverse_brain',
  game_mode VARCHAR(64) NOT NULL DEFAULT 'classic',
  category VARCHAR(32) NOT NULL,
  title VARCHAR(64) NOT NULL,
  description VARCHAR(255) NOT NULL,
  icon VARCHAR(64) NOT NULL DEFAULT 'face-pink-wink',
  reward_type VARCHAR(16) NOT NULL DEFAULT 'coin',
  reward_amount INT UNSIGNED NOT NULL DEFAULT 0,
  target_value DECIMAL(12, 2) NOT NULL DEFAULT 1,
  condition_key VARCHAR(64) NOT NULL,
  sort_order INT NOT NULL DEFAULT 0,
  is_hidden TINYINT(1) NOT NULL DEFAULT 0,
  is_enabled TINYINT(1) NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_achievement_key (achievement_key),
  KEY idx_game_category (game_key, game_mode, category, sort_order)
);

CREATE TABLE user_achievements (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY,
  user_id VARCHAR(64) NOT NULL,
  achievement_id BIGINT UNSIGNED NOT NULL,
  current_value DECIMAL(12, 2) NOT NULL DEFAULT 0,
  unlocked TINYINT(1) NOT NULL DEFAULT 0,
  unlocked_at DATETIME NULL,
  reward_claimed TINYINT(1) NOT NULL DEFAULT 0,
  claimed_at DATETIME NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_user_achievement (user_id, achievement_id),
  KEY idx_user_unlocked (user_id, unlocked),
  CONSTRAINT fk_user_achievements_definition
    FOREIGN KEY (achievement_id) REFERENCES achievement_definitions(id)
);
```

## 后端接口实现思路

伪代码：

```ts
app.get("/api/achievements", async (req, res) => {
  const { userId, gameKey = "reverse_brain", gameMode = "classic" } = req.query;

  if (!userId) {
    return res.json({ code: 10001, message: "userId is required" });
  }

  const definitions = await db.query(`
    SELECT *
    FROM achievement_definitions
    WHERE game_key = ?
      AND game_mode = ?
      AND is_enabled = 1
    ORDER BY sort_order ASC, id ASC
  `, [gameKey, gameMode]);

  const progressRows = await db.query(`
    SELECT *
    FROM user_achievements
    WHERE user_id = ?
  `, [userId]);

  const progressMap = new Map(progressRows.map(row => [row.achievement_id, row]));

  const achievements = definitions.map(def => {
    const progress = progressMap.get(def.id);
    const current = Number(progress?.current_value ?? 0);
    const target = Math.max(1, Number(def.target_value));
    const unlocked = Boolean(progress?.unlocked);

    return {
      id: def.achievement_key,
      category: def.category,
      title: def.title,
      description: def.description,
      icon: def.icon,
      rewardKind: def.reward_type,
      rewardAmount: Number(def.reward_amount),
      current,
      target,
      progressText: `${Math.min(current, target)}/${target}`,
      unlocked,
      showProgress: !unlocked
    };
  });

  res.json({
    code: 0,
    message: "ok",
    data: {
      unlockedCount: achievements.filter(item => item.unlocked).length,
      totalCount: achievements.length,
      achievements
    }
  });
});
```

## 和游戏总结接口联动

已有游戏总结上传接口：

```http
POST /api/game-scores/submit
```

成就进度更新建议直接写在这个接口的后端流程里，不建议前端在游戏结束后再单独调用一个“更新成就”接口。

推荐后端流程：

1. 校验 `userId`、`gameKey`、`gameMode` 和本局结果字段。
2. 用 `clientRoundKey` 做幂等检查，避免同一局重复上传导致累计类成就重复增加。
3. 保存本局成绩到游戏记录表。
4. 在同一个事务里调用 `updateAchievementsByGameSummary` 更新 `user_achievements`。
5. 返回本局成绩结果，同时可返回本次变化的成就。

前端当前已经配合：

- `GameSummaryDisplay` 会在 `/api/game-scores/submit` 请求体里带上 `clientRoundKey`。
- `/api/game-scores/submit` 成功后，前端会标记成就缓存过期。
- 玩家下次打开成就页时，会自动请求 `/api/achievements` 刷新进度。

请求体里和成就计算相关的字段：


前端目前不强依赖 `achievementsChanged` 和 `unlockedAchievements`，但建议后端先返回，后续做“游戏总结页弹出新解锁成就”时可以直接用。

当前 6 个成就的计算规则：

| achievement_key | 触发/进度来源 | 更新方式 |
| --- | --- | --- |
| `tutorial` | 用户完成教程；如果暂时没有教程事件，可在首次有效成绩上传后置为 1 | 固定写 1 |
| `ten_streak` | 本局 `maxCombo` | 取历史最大值，达到 10 解锁 |
| `perfect_round` | 本局 `accuracy >= 100` 且 `totalQuestions > 0` | 满足写 1 |
| `combo_master` | 本局 `maxCombo` | 取历史最大值，达到 50 解锁 |
| `reverse_master` | 累计游戏时长 | 累加有效局 `durationMs`，换算小时，达到 10 解锁 |
| `cold_brain` | 困难难度单局 0 失误 | `difficulty = hard` 且 `wrongCount = 0` 且 `missedCount = 0` 时写 1 |

建议后端实现：

```ts
async function updateAchievementsByGameSummary(userId: string, summary: GameSummaryPayload) {
  const changedAchievements = [];

  changedAchievements.push(
    await setAchievementProgress(userId, "tutorial", 1, "max")
  );

  changedAchievements.push(
    await setAchievementProgress(userId, "ten_streak", summary.maxCombo, "max")
  );

  changedAchievements.push(
    await setAchievementProgress(userId, "perfect_round", summary.accuracy >= 100 && summary.totalQuestions > 0 ? 1 : 0, "max")
  );

  changedAchievements.push(
    await setAchievementProgress(userId, "combo_master", summary.maxCombo, "max")
  );

  const totalDurationMs = await getUserTotalDurationMs(userId, summary.gameKey, summary.gameMode);
  changedAchievements.push(
    await setAchievementProgress(userId, "reverse_master", totalDurationMs / 3600000, "replace")
  );

  changedAchievements.push(
    await setAchievementProgress(
      userId,
      "cold_brain",
      summary.difficulty === "hard" && summary.wrongCount === 0 && summary.missedCount === 0 ? 1 : 0,
      "max"
    )
  );

  return changedAchievements.filter(Boolean);
}
```

`setAchievementProgress` 规则：

- 找到 `achievement_definitions.achievement_key`
- upsert 到 `user_achievements`
- 最大值类成就用 `GREATEST(current_value, newValue)`
- 累计类成就建议由成绩表汇总出总值后 `replace` 写入，避免重复上传时重复累加
- `current_value >= target_value` 时设置 `unlocked = 1` 和 `unlocked_at`
- 已解锁成就不要重复覆盖 `unlocked_at`
- 函数最好返回“本次是否从未解锁变为已解锁”，方便接口返回 `unlockedAchievements`

```ts
async function setAchievementProgress(
  userId: string,
  achievementKey: string,
  currentValue: number,
  mode: "max" | "replace"
) {
  const def = await findAchievementDefinition(achievementKey);
  if (!def) return null;

  const nextValue = Math.max(0, currentValue);
  const unlocked = nextValue >= Number(def.target_value);
  const oldProgress = await findUserAchievement(userId, def.id);
  const wasUnlocked = Boolean(oldProgress?.unlocked);

  await db.query(`
    INSERT INTO user_achievements (
      user_id,
      achievement_id,
      current_value,
      unlocked,
      unlocked_at
    ) VALUES (?, ?, ?, ?, ?)
    ON DUPLICATE KEY UPDATE
      current_value = IF(? = 'replace', VALUES(current_value), GREATEST(current_value, VALUES(current_value))),
      unlocked = IF(
        unlocked = 1,
        1,
        IF(IF(? = 'replace', VALUES(current_value), GREATEST(current_value, VALUES(current_value))) >= ?, 1, 0)
      ),
      unlocked_at = IF(
        unlocked_at IS NOT NULL,
        unlocked_at,
        IF(IF(? = 'replace', VALUES(current_value), GREATEST(current_value, VALUES(current_value))) >= ?, CURRENT_TIMESTAMP, NULL)
      ),
      updated_at = CURRENT_TIMESTAMP
  `, [
    userId,
    def.id,
    nextValue,
    unlocked ? 1 : 0,
    unlocked ? new Date() : null,
    mode,
    mode,
    Number(def.target_value),
    mode,
    Number(def.target_value)
  ]);

  const isNewUnlock = !wasUnlocked && unlocked;
  return {
    id: def.achievement_key,
    title: def.title,
    changed: true,
    unlocked: isNewUnlock
  };
}
```

## 后续新增成就流程

1. 在 `achievement_definitions` 插入新成就定义。
2. 后端在登录、教程完成、游戏总结上传等节点更新 `user_achievements`。
3. 成就页无需改代码，接口返回新数据后自动展示。
4. 如果新成就需要新图标，把图片放到 `assets/resources/textures/achievements`，并让接口 `icon` 返回对应 SpriteFrame 名称。
